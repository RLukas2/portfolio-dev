# Website To-Do List

This is a living document tracking planned features and improvements for my personal portfolio.
I'll update this as I progress through each item.

## Get Started

I have long wanted to create a personal website that serves as a central hub for my work, projects, and blog posts.
My goal is to build a platform that is easy to navigate, visually appealing, and simple to maintain.
I am using [Next.js](https://nextjs.org/), [Supabase](https://supabase.com/), and [Prisma](https://www.prisma.io/) to build this website, leveraging their powerful features to create a seamless user experience.

## Initial Setup

- [x] Set up Next.js project with TypeScript
- [x] Integrate Tailwind CSS for styling
- [x] Configure Supabase for backend services
- [x] Set up Prisma for database management
- [x] Implement authentication with Supabase Auth
- [x] Design and implement the database schema
- [x] Create initial pages: Home, About, Projects, Blog, Contact
- [x] Deploy the website on Vercel

One of the reasons I deploy on Vercel rather than Github Pages is the built-in support for serverless functions and API routes, which allows me to create a more dynamic and interactive experience for users.

Another reason is managing static and dynamic content more efficiently, as Vercel optimizes the delivery of both types of content seamlessly; meanwhile, Github Pages is primarily designed for static content.

## Why Improve?

As I continue to develop my website, I want to ensure that it remains up-to-date with the latest web development trends and technologies.
By continuously improving the website, I can provide a better user experience and showcase my skills as a developer.

And some of the aesthetic elements feel a bit outdated, not fit, or just plain boring.
That is why I want to keep pushing the boundaries and exploring new design trends to make my website more visually appealing.
